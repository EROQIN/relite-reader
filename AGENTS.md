# Relite Reader Agent Instructions

## Project Overview
Relite Reader is a full-stack web reading app with user login, WebDAV-backed personal libraries, multi-format ebook reading (EPUB, PDF, MOBI, CBZ, and more), and customizable reading themes. When a user is logged in, reading progress and preferences are persisted.

## MVP Scope
- Email + password auth with JWT.
- Add/manage WebDAV storage, index library metadata, and read by streaming files.
- Reader UI supports multi-format rendering and per-user preferences (theme, font, background, spacing).
- Reading progress synced to the backend.
- Mobile-first PWA (installable web app, offline shell, responsive UI).

## Tech Stack
- Backend: Go, REST/JSON APIs, JWT auth.
- Frontend: React + Vite, PWA build.
- Database: PostgreSQL + pgvector extension.
- Storage: WebDAV streaming with optional caching policy.

## Minimal Architecture
- Single Go service serves API and static frontend assets.
- Background sync crawls WebDAV directories and updates the book index.
- Pluggable parser interface for formats (start with EPUB/PDF, extend later).
- Optional cache for parsed metadata/content segments (bounded size).
- Future LLM features use pgvector embeddings without changing core flows.

## Data Flow
1. User logs in and adds a WebDAV connection.
2. Backend validates credentials and stores them encrypted at rest.
3. Background sync indexes remote files into book metadata records.
4. User opens a book; backend streams and parses content on demand.
5. Frontend stores UI state locally and persists progress/preferences via API.

## Data Model (Minimum)
- users: email, password hash, created_at.
- webdav_connections: user_id, base_url, username, encrypted_secret.
- books: user_id, title, author, format, source_path, metadata_json.
- reading_progress: user_id, book_id, location, updated_at.
- user_preferences: user_id, theme, font, background, spacing.
- embeddings (future): user_id, book_id, vector, metadata.

## Security & Privacy
- Never log credentials or tokens.
- Encrypt WebDAV secrets at rest with a server key.
- Enforce per-user ownership on all queries and file access.
